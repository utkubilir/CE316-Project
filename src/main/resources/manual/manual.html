CE316 Integrated Assignment Environment

Lightweight lecturer-side grading tool for ZIPped student submissions.

Overview

The IAE compiles, runs, and grades a folder of student ZIP files against a single expected output. It supports compiled languages (C, C++, Java) and interpreted ones (Python). All project data is stored locally in iae_projects.db; no server is required.

Quick Start

  1. File → New Project, enter a name, choose or create a configuration.
  2. Pick the folder containing student ZIP files (sidebar Browse...).
  3. Pick the expected output file.
  4. Click Run Tests.
  5. Inspect the results table; export an HTML report via File → Export Report.

Projects

  • New Project — starts a fresh evaluation session. Asks for a name and configuration.
  • Open Project — reloads a saved project including its results.
  • Save Project — persists name, configuration, submission folder, expected output path, and the latest results into SQLite.
  • Delete Project — removes the project and its saved results.

Configurations

A configuration describes how to compile and run a particular language. The fields are:

FieldDescription
NameUnique identifier (e.g. C Configuration).
LanguageUsed as a hint and label (e.g. C, Java).
Source FileFilename the IAE looks for inside the extracted ZIP.
Compile CommandShell command run before the program. Leave blank for interpreted languages.
Run CommandHow to invoke the compiled or interpreted program.
Run ArgsCommand-line arguments appended after the run command. Quoted strings stay together (e.g. apple "banana split").

Manage Configurations

Open Configuration → Manage Configurations... to create, edit, delete, or export saved configurations. Export writes the selected configuration to JSON. Export All bundles every saved configuration into one JSON array — perfect for sharing or backing up.

Import & Export

File → Import Configuration(s) accepts either a single configuration JSON file or a JSON array; all imported entries are saved. File → Export Current Form to JSON exports whatever is currently entered in the Configuration tab (useful before saving).

Student Submissions

  • Each student must submit a single .zip file. The ZIP filename (without extension) is treated as the student ID.
  • Inside the ZIP, the source file may sit at the top level or in a nested folder — the IAE searches recursively for the configured source file name (case-insensitive).
  • Use the sidebar Browse... button to pick the folder, then Scan (or auto-scan on selection) to preview the discovered ZIPs.
Files are extracted into a temporary working_directory/ at the application root, cleaned up automatically at the start of each run.

Expected Output

  • Provide a .txt or .out file containing the program's expected stdout.
  • The Expected Output tab loads the file's contents into an editor where it can be tweaked and saved back.
  • Output comparison is whitespace-normalized: CRLF/LF differences, leading/trailing whitespace, and runs of spaces or blank lines are collapsed before comparison.

Running Tests

  • Click Run Tests in the sidebar. The progress bar and status bar update per submission.
  • Each student program is killed after 10 seconds to prevent infinite loops; the result is recorded as Timeout.
  • Cancel Run stops processing after the current submission finishes.

Reading Results

The results table shows one row per student with a colored status pill:

PillMeaning
PassedOutput matches the expected output.
Output MismatchProgram ran successfully but output differs.
Compile ErrorCompile command failed; compiler output captured in details.
Runtime ErrorProgram returned non-zero exit code.
TimeoutProgram exceeded the 10-second budget.
Missing SourceConfigured source file was not found in the ZIP.
Extraction ErrorZIP could not be opened (corrupted, password-protected).

Use the Filter dropdown above the table to narrow by status. Double-click a row (or hit the Details button) to see the full compiler/runtime output for that student.

Exporting Reports

File → Export Report (or the button above the results table) writes a self-contained HTML report with: project context, configuration, summary counts, and per-student rows including full compiler/runtime output. Open the file in any browser. The path is remembered on the project so the next export defaults to the same folder.

Supported Languages

The four built-in defaults (C, C++, Java, Python) are seeded on first launch. Any other language can be supported by creating a custom configuration — the IAE only needs the compile and run commands; the rest of the toolchain (gcc, javac, python, etc.) is invoked through the OS.

Troubleshooting

  • "Source file not found" — verify the source file name matches the ZIP contents exactly (case-insensitive). The IAE searches recursively.
  • "Compile Error" pill — open Details; the compiler's stdout/stderr is captured verbatim.
  • Timeout — increase the program's efficiency or ensure no scanf is blocking; the IAE always passes through stdin via the run args, not stdin.
  • Mismatch on whitespace only — confirm the expected output file has no trailing blank lines; the IAE trims and normalizes whitespace before comparing.
  • "Extraction Error" — re-zip the submission with a standard tool; password-protected ZIPs are unsupported.
